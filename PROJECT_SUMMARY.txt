PROJECT OVERVIEW
================

TokenShield is a full-stack token security analysis platform that helps
traders and investors identify safe tokens and avoid scams on Solana and
BNB Smart Chain.

QUICK START
===========

1. Terminal 1 - Start Backend:
   cd backend
   npm install
   npm start
   
   → Runs on http://0.0.0.0:3000

2. Terminal 2 - Start Frontend:
   npm install
   npm run dev
   
   → Opens http://localhost:5173

3. Open browser and scan a token!

AVAILABLE ENDPOINTS
===================

POST /api/scan
  • Main scanning endpoint
  • Analyzes token security & risk
  • Request: {"token_address":"...", "chain":"solana/bsc"}
  • Response: Complete analysis report

GET /api/history
  • Retrieve last 50 scanned tokens
  • Shows cached scan results
  • No parameters needed

GET /api/token/:address/:chain
  • Get specific token details
  • Returns cached analysis
  • Params: address, chain

GET /api/health
  • Health check endpoint
  • Returns: {"status":"OK"}

RISK SCORING SYSTEM
===================

Risk Score (0-100):
  1-29:   Very Low Risk   ✅ Strong entry
  30-44:  Low Risk        🟢 Good opportunity
  45-59:  Medium Risk     🟡 Moderate caution
  60-74:  High Risk       🟠 Caution advised
  75-100: Critical Risk   🔴 Avoid

Entry Signals:
  Strong Buy  🚀 Signal Score ≥50 (excellent)
  Buy         📈 Signal Score 25-49 (good)
  Moderate    ⏸️  Signal Score 0-24 (neutral)
  Avoid       ❌ Signal Score <0 (dangerous)

ANALYSIS FACTORS
================

Security (40% weight):
  • Ownership renounced or active
  • Mint function enabled/disabled
  • Honeypot status
  • Freeze authority
  • Rug pull risk assessment

Market (40% weight):
  • Market capitalization
  • Liquidity availability
  • Trading volume
  • Price momentum
  • 24h price changes

Holders (20% weight):
  • Total holder count
  • Top 10 concentration
  • Distribution health
  • Whale holder presence

PRODUCTION DEPLOYMENT
=====================

Before deploying:

1. Environment Setup:
   • Set BSCSCAN_API_KEY for enhanced BSC data
   • Update VITE_API_URL to production URL
   • Configure NODE_ENV=production

2. Security Hardening:
   • Add rate limiting (express-rate-limit)
   • Implement API key authentication
   • Add HTTPS/TLS encryption
   • Set secure CORS policy
   • Add request validation

3. Database:
   • Migrate to PostgreSQL for scale
   • Set up regular backups
   • Add connection pooling
   • Implement query caching

4. Monitoring:
   • Add logging (Winston/Morgan)
   • Set up error tracking (Sentry)
   • Monitor API response times
   • Track RPC endpoint health

5. Hosting Options:
   • Frontend: Vercel, Netlify
   • Backend: Railway, Render, AWS
   • Database: Managed PostgreSQL
   • CDN: Cloudflare
